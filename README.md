# AI Enablement Lab 🧪

**Where a Chief Strategy & Data Officer builds his AI claims by hand — in public.**

I've spent 15 years telling the agri-inputs industry what global change means on the ground: regulation, supply shocks, M&A, geopolitics, value-chain shifts. Today I lead strategy, data and AI programs at the industry's market-intelligence leader. That work is synthesis and judgment at scale — which makes it the perfect testbed for AI done right.

This repo is my lab. Every project here follows the same standard, which I call **vibe engineering**:

> Vibe *coding* is prompting your way to a demo that works once.
> Vibe *engineering* is building fast with AI while adding the things that make it survive reality: version control, evals, error handling, monitoring, security, documentation, and a rollback plan.

## How every project is documented

Each project folder contains a case-study README:
**Problem → Users → Approach & architecture → What broke and how I fixed it → Results (time/cost numbers) → What I'd do differently.**
If a project has no numbers yet, it says so. No hype.

## Principles this lab runs on

1. Start with the workflow, not the model.
2. A demo is a hypothesis, not a product.
3. If you can't measure it, you shipped a vibe.
4. Adoption eats accuracy for breakfast.
5. Governance is a feature — it's what lets you ship in regulated industries.
